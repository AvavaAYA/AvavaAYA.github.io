File Stream Oriented Programming:

  1. 劫持_IO_list_all的值伪造_IO_FILE项
  2. 调用_IO_flush_all_lockp刷新_IO_list_all链表中所有项的文件流，相当于调用_IO_FILE_plus.vtable中的_IO_overflow:

对于我们想触发调用_IO_OVERFLOW，需要满足:

即:

  • fp->_mode <= 0 && fp->_IO_write_ptr > fp->_IO_write_base

或者:

  • _IO_vtable_offset (fp) == 0
  • fp->_mode > 0
  • (fp->_wide_data->_IO_write_ptr > fp->_wide_data->_IO_write_base)

一般选择伪造前者.

_IO_flush_all_lockp 函数在一些情况下会被系统调用:

  • exit()
  • libc执行abort流程
  • 执行流从main函数返回
